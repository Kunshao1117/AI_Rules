# Audit Denominator And Generated Artifact Authority Policy

This reference defines which sources may be used as current denominators,
completion evidence, or governance authority when audit, memory, generated
context, log, and cache artifacts disagree.

## Authority Boundary

- Canonical runtime governance lives in source policies, source skills,
  platform cores, scripts, test runners, and official tool output from the
  current run.
- Memory cards are durable source facts, status pointers, owner pointers, and
  attribution aids. They are not runtime governance rule sources.
- Generated logs, generated AI-context packs, generated snapshots, and cache
  indexes are non-authoritative evidence artifacts. They may point to evidence
  but cannot replace current source, runner output, or a fresh manifest.

## Memory Freshness

- `status: stale`, `staleness > 0`, or a freshness warning overrides any
  current-verification reading of `verification_status: verified`.
- `verification_status: verified` means the card was verified as of
  `last_verified` or `verified_as_of`; it does not prove current validity after
  tracked source changes.
- When both verified and stale evidence exist, report the fact as
  `verified_as_of` plus `current_validity: stale_source_pending_review`,
  `partial_evidence`, or another explicit non-current state.
- Staleness warnings are evidence freshness warnings only. They must not be
  phrased as standalone hard behavior rules. Agents still follow canonical
  source policies, skills, and current source files.

## Fixture And Denominator Counts

- Fixture totals, issue denominators, tracked-file totals, and coverage counts
  must be computed from the current runner, manifest, or live file inventory at
  validation time.
- Memory prose may describe ownership and grouping, but it must not provide the
  authoritative current fixture count.
- Historical counts inside archives remain historical only and must be labeled
  or treated as non-current.

## `.cfce/ai-context/**`

- Files under `.cfce/ai-context/**` are generated snapshots for AI handoff.
- Statements such as zero issues, no actionable items, or complete task packs
  are scoped to the recorded `workspaceRoot`, `generatedAt`, scan mode,
  command set, and tool capability surface in that snapshot.
- The TTL is until the next CFCE scan, source change affecting the denominator,
  tool capability change, or 24 hours after `generatedAt`, whichever comes
  first.
- These files are non-authoritative for current completion, current issue
  count, audit readiness, or release readiness. Current decisions require a
  fresh CFCE scan or project validation command output.

## `.cartridge/index.json`

- `.cartridge/index.json` is a cache generated by the cartridge/memory tooling.
- Fields such as `status`, `contentQualityStatus`, `verificationStatus`,
  `staleness`, `pendingChanges`, `ghostFiles`, and tracked-file indexes are
  cache-only facts from the last scan.
- The index is not a current decision source for completion, memory freshness,
  issue denominators, or governance authority.
- The TTL is until the next memory scan/reindex, source change affecting a
  tracked file, memory-card edit, or 24 hours after the index `lastScanned`,
  whichever comes first.
- Current decisions must read the active memory card, current source files, and
  fresh read-only memory tool or runner output.

## `.agents/logs/**`

- Files under `.agents/logs/**` are task evidence, archived helpers, or
  generated utilities unless a current source policy explicitly promotes a file
  to source ownership.
- Logs-path utilities such as `.agents/logs/generate_cards.ps1` are archived
  generated utilities. They must not define current memory schema, topology, or
  governance behavior.
- If a logs-path helper remains referenced for compatibility, keep the existing
  reference stable and document the canonical source owner instead of moving or
  rewriting references opportunistically.

## Decision Rule

When a generated/cache/log artifact conflicts with source, runner output, or a
fresh manifest, use the source, runner output, or fresh manifest. Report the
generated/cache/log artifact as stale, scoped, non-authoritative, or
informational.
