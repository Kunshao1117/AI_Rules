# Antigravity 更新說明

## v6.0.2 — 2026-04-04

- **跨語系實體武裝檢核**：在 `cross-lingual-guard` 強行插入 Phase 2 檢查，中斷意圖解析後的盲猜，強制映射實體技能或 MCP 工具，根除大語言模型的零樣本惰性。
- **文件防腐防線 (Doc-Sync Guard)**：將完成閘門中無約束力的黃燈提醒升級為紅色硬閘門；同時將公共關聯文件納入變更影響分析 (`impact-test-strategy`)，徹底消滅程式碼推進但文件過期的死角。
- **部署防污機制**：修復 Deploy 腳本 Fresh 模式因暴力複製導致 `memory` 污染的問題，改以源頭過濾與動態生成空殼結構。

## v6.0.1 — 2026-04-04

- **全域版本控制鎖定**：新增 `_system` 記憶卡，強制將後續的 Git 操作根目錄定錨在母資料夾，實現零手動控制的背景備份防線。

## v6.0.0 — 2026-04-03

- **三位一體治理範式**：全框架 23 支核心組件從「教學指引」模式全面升級為「靜默異常中斷 + 特權覆寫 + 雙軌沙盒」三合一指令化控制。AI 在正常運作時零輸出，只在閘門攔截時才以紅字中斷回報。
- **靜默異常中斷閘門**：22 支檔案植入決策樹式閘門（`[SILENT GATE]`），涵蓋終端機寫入、機密掃描、驗證器、記憶卡同步、單元測試、回歸測試、DOM 選擇器、Schema 驗證、影響分析、PR 合併等全生命週期。
- **特權覆寫機制（SUDO）**：總監在指令句首附加 `[SUDO]` 即可無條件繞過所有閘門（`/03_build` 除外 — 正規建構嚴禁破窗）。19 支組件支援特權覆寫。
- **沙盒實驗工作流（/03_sketch）**：全新建立的第 13 道工作流。所有品質、安全、測試、記憶閘門全數停用，用於快速原型實驗與髒扣探索。
- **斷路器機制**：`/03_build`、`/04_fix` 植入最大重試次數限制（3 次 / 2 次），防止 AI 無限迴圈自我修復。超過上限則硬性中斷。
- **狀態保全閘門**：`/05_refactor` 植入基線測試比對，重構前後結果不一致自動退版。新增「無測試框架」邊界分支處理。
- **三位一體基因遺傳**：`/12_skill_forge` 與 `skill-factory` 植入遺傳機制，所有新產生的衍生技能自動繼承靜默閘門與特權覆寫 DNA。
- **文件自動同步閘門**：完成閘門新增 Check 7（記憶卡驅動，無寫死檔名），提交工作流升級為 `[DOC SYNC GATE]`，框架檔案修改後自動提醒更新系統文件。
- **工作流數量**：12 → 13 道（新增 `/03_sketch`）

## v5.6.0 — 2026-04-03

- **跨語系防護規則獨立化**：將跨語系思維防護協議從 `00_core_identity.md` 拆出，建立獨立的 `01_cross_lingual_guard.md`（Always On）。規則檔短小精悍（55 行），大幅提升新對話 AI 的冷啟動合規率。
- **冷啟動強制讀檔機制**：全新對話的第一次非平凡中文輸入，強制要求 AI 先執行 `view_file` 讀取跨語系防護技能（工具動作，不可跳過），確保防護協議被機械性激活而非僅靠「記得」。
- **移除信心閥值**：廢除 AI 自評信心的 HIGH/MEDIUM/LOW 機制（自評不可靠，高信心時正是最危險的時刻），改為透過 `<details>` 模板永遠輸出供總監直接確認。
- **部署腳本 Fresh 模式記憶保護**：`Deploy-Antigravity.ps1` Fresh 模式新增備份還原機制，部署前先備份 `memory/` 與 `project_skills/`，複製完成後立即還原，防止記憶卡與衍生技能被強制覆蓋。
- **Turbo 安全攔截閘門**：在 `_security_footer.md` 新增毀滅性指令黑名單（`reset --hard`、`rm -rf`、`DROP TABLE` 等），即使工作流標記 `// turbo-all` 也強制要求總監確認。
- **衍生技能命名前綴強制化**：`skill-factory` 新增「衍生技能必須使用 `{project-code}-{skill-name}` 格式」規範，消除與框架核心技能的命名碰撞風險。
- **死鎖計數器持久化**：`delegation-strategy` 新增 `.gemini/validation_state.json` 狀態檔機制，驗證失敗計數不再依賴 AI 短期記憶，確保熔斷機制在長對話中仍能正確觸發。
- **時區強制物理校正**：`memory-ops` 新增台灣時間強制轉換的 PowerShell 指令，確保記憶卡時間戳在任何系統時區設定下均輸出 `+08:00`。
- **CLI 寫入豁免條款**：`00_core_identity.md` §5 新增 CLI 子代理人的 `.agents/logs/` 寫入豁免，解除 CLI 分析任務無法回傳結果的死鎖。
